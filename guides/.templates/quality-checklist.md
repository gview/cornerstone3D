# 文档质量检查清单

**Purpose**: 验证文档是否符合 Cornerstone3D 开发指南的质量标准
**Version**: 1.0.0
**Date**: 2025-01-18

---

## 使用说明

在发布任何文档之前，必须使用本检查清单验证文档质量。所有项目必须标记为完成（✅）才能发布。

---

## 1. 语言和格式

### 中文优先

- [ ] 文档使用中文编写
- [ ] 技术术语首次出现时提供中文解释（格式：`英文术语（中文解释）`）
- [ ] 后续可继续使用英文术语，无需重复解释
- [ ] 示例：`Viewport（视口）`、`RenderingEngine（渲染引擎）`

### 格式规范

- [ ] 文档通过 Prettier 格式化
- [ ] 没有尾随空格
- [ ] 使用正确的标题层级（h2-h4）
- [ ] 列表格式正确（无序列表或有序列表）
- [ ] 代码块使用正确的语言标识符

### 标题规范

- [ ] 标题清晰明确，避免模糊
- [ ] 标题层级合理（不超过 4 级）
- [ ] 标题使用中文
- [ ] 避免使用特殊字符（除了必要的标点）

---

## 2. Frontmatter 元数据

- [ ] 包含完整的 frontmatter 块
- [ ] `id` 字段存在且唯一（kebab-case 格式）
- [ ] `title` 字段存在且为中文
- [ ] `category` 字段存在且为有效值（architecture、getting-started、advanced、examples、troubleshooting）
- [ ] `order` 字段存在且为正整数
- [ ] `description` 字段存在（1-2 句话）
- [ ] `prerequisites` 字段存在（可为空数组）
- [ ] `estimatedTime` 字段存在（格式："[数字] [单位]"）
- [ ] `difficulty` 字段存在（beginner、intermediate、advanced）
- [ ] `tags` 字段存在（数组，可为空）

---

## 3. 内容质量

### 清晰性

- [ ] 内容清晰易懂，逻辑连贯
- [ ] 步骤明确，避免歧义
- [ ] 没有拼写错误或语法错误
- [ ] 专业术语使用正确

### 完整性

- [ ] 覆盖主题的所有关键方面
- [ ] 提供必要的背景信息
- [ ] 包含完整的代码示例（如果适用）
- [ ] 包含预期结果或输出说明

### 准确性

- [ ] 技术信息准确无误
- [ ] 代码示例经过验证
- [ ] API 引用正确
- [ ] 版本信息正确

---

## 4. 代码示例

### 代码质量

- [ ] 代码包含完整的中文注释
- [ ] 代码格式化（Prettier）
- [ ] TypeScript 代码通过类型检查（`tsc --noEmit`）
- [ ] 代码可以直接运行或仅需最小修改
- [ ] 变量名和函数名有意义

### 代码注释

- [ ] 关键步骤有中文注释
- [ ] 复杂逻辑有详细说明
- [ ] 错误处理有注释说明
- [ ] TODO/FIXME 注释合适使用

### 代码块格式

- [ ] 指定正确的语言标识符
- [ ] 包含文件路径（如果适用）
- [ ] 必要时高亮关键行
- [ ] 代码块有说明文字

---

## 5. 链接和引用

### 内部链接

- [ ] 所有内部链接使用相对路径
- [ ] 链接文本描述清晰
- [ ] 链接有效（指向存在的文档）
- [ ] 锚点链接正确（如果有）

### 外部链接

- [ ] 外部链接可访问
- [ ] 链接描述清晰
- [ ] 必要时添加 `rel="nofollow"`
- [ ] 引用的外部资源权威可靠

### 交叉引用

- [ ] 相关文档有适当的交叉引用
- [ ] 使用 `See also` 或类似提示
- [ ] 前置和后续文档链接正确

---

## 6. 视觉元素

### 图片和图表

- [ ] 图片包含 `alt` 属性（中文描述）
- [ ] 图片大小合适（不超过 500KB）
- [ ] 图表使用 Mermaid.js 格式
- [ ] 图表有说明文字
- [ ] 图表在支持的渲染器中正确显示

### 表格

- [ ] 表格有表头
- [ ] 表格对齐整齐
- [ ] 表格有说明（如果复杂）
- [ ] 表格数据准确

### 列表

- [ ] 列表格式一致
- [ ] 列表项完整
- [ ] 嵌套列表层级清晰

---

## 7. 示例和实践

### 示例代码

- [ ] 提供可运行的完整示例
- [ ] 示例代码有预期的输出说明
- [ ] 示例代码符合最佳实践
- [ ] 示例代码易于理解和修改

### 实践步骤

- [ ] 步骤编号清晰
- [ ] 每个步骤可独立验证
- [ ] 包含验证步骤
- [ ] 提供故障排查提示

---

## 8. 文档结构

### 章节组织

- [ ] 包含必要的章节（简介、主要内容、总结等）
- [ ] 章节顺序合理
- [ ] 章节之间有良好的过渡
- [ ] 使用目录（TOC）如果文档较长

### 导航

- [ ] 有清晰的文档导航
- [ ] 包含"下一步"或相关文档链接
- [ ] 有"返回"或"上级"链接（如果适用）

---

## 9. 可访问性

### 可读性

- [ ] 文本对比度足够
- [ ] 字体大小合适
- [ ] 行距和段落间距适当

### 可理解性

- [ ] 避免过度使用技术术语
- [ ] 必要术语有解释
- [ ] 使用类比和示例帮助理解
- [ ] 考虑不同技能水平的读者

## 10. 版本和维护

### 版本信息

- [ ] 文档包含版本信息（如果适用）
- [ ] 更新日期准确
- [ ] 与代码版本同步

### 维护性

- [ ] 文档易于更新
- [ ] 避免硬编码的版本号（除非必要）
- [ ] 避免硬编码的路径
- [ ] 使用相对路径

- [ ] 包含最佳实践

## 质量标准总结

### 必须满足的条件（GATE）

文档发布前**必须**满足以下所有条件：

1. ✅ 中文编写，技术术语有中文解释
2. ✅ 包含完整的 frontmatter 元数据
3. ✅ 代码示例有中文注释
4. ✅ 代码通过 Prettier 格式化和 TypeScript 类型检查
5. ✅ 所有链接有效
6. ✅ 包含验证步骤或预期结果说明

### 推荐满足的条件

建议满足但不强制：

- 包含图表或示意图
- 提供多个示例
- 包含故障排查部分
- 包含性能优化建议

---

## 质量等级

| 等级 | 描述 | 要求 |
|------|------|------|
| **A** | 优秀 | 满足所有必须和推荐条件 |
| **B** | 良好 | 满足所有必须条件 |
| **C** | 合格 | 满足大部分必须条件，有少数待改进 |
| **D** | 不合格 | 不满足多个必须条件，需要改进 |

**发布标准**: 文档质量必须达到 **B 级或以上**才能发布。

---

## 检查流程

### 自查（作者）

1. 完成文档编写
2. 使用本检查清单自查
3. 修复发现的问题
4. 标记所有项目为完成（✅）

### 审查（审核者）

1. 阅读文档
2. 使用本检查清单审查
3. 提供反馈和改进建议
4. 确认质量等级

### 批准（维护者）

1. 检查质量等级
2. 确认满足发布标准
3. 批准发布
4. 记录版本信息
